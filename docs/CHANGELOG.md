# Changelog

## 1.3.0

### Added

- グループを一時停止とは別に無効化できる機能を追加しました。
- Options 画面のグループカードにアクションメニューを追加し、一時停止、無効化、削除の操作を整理しました。
- Popup で対象グループのブロック状態、一時停止状態、ブロック時間帯、日別上限の状態を表示できるようにしました。
- 一時停止の待機中、準備完了、一時停止中を Popup 表示向けに判定するユーティリティを追加しました。

### Changed

- 無効化されたグループを URL 判定、counter、通知、redirect URL 判定の対象から除外するようにしました。
- Options 画面のグループ状態表示に Disabled badge と無効化状態の要約を追加しました。
- Popup の空状態文言を、日別上限だけでなく有効な制限全体を表す内容に変更しました。
- Daily limit の表示を調整し、上限が設定されている場合だけ `min/day` を表示するようにしました。
- 設定エクスポートのスキーマバージョンを `4` に更新し、`2` と `3` の import 互換を維持しました。

### Fixed

- 無効化グループが閲覧時間 counter や最小残り時間の判定に残らないようにしました。
- グループ無効化、一時停止表示、Popup 状態表示、ブロック判定、ストレージ互換のテストを追加・更新しました。

## 1.2.0

### Added

- グループ単位でブロックを一時停止できる機能を追加しました。
- 一時停止リクエスト後にカウントダウンダイアログを表示し、待機完了後に一時停止を開始できるようにしました。
- 一時停止中のグループに残り時間を表示するようにしました。
- `groupPauseState` の保存仕様を追加し、期限切れや不正な runtime 状態を安全に除外するようにしました。
- 通知計画のユーティリティを追加し、残り時間通知、対象ページを開いた時の通知、redirect ブロック通知の判定を整理しました。
- 仕様ドキュメントを `docs/spec/` 配下に整理し、ストレージ、UI、権限、非要件の記述を追加しました。

### Changed

- Options 画面のグループカード、グローバル設定、URL pattern、Daily limit、Blocked time、通知設定の表示文言をより明確にしました。
- グループ編集時の Options 項目を折りたたみ表示にし、通常表示では重要な設定だけを要約するようにしました。
- URL pattern の placeholder を具体的な例に変更しました。
- 空の URL patterns 状態でも追加ボタンを表示するようにしました。
- Active settings とグループカードに読み取り専用表示を追加しました。
- 一時停止リクエストのボタンラベルを `Request pause` に変更しました。
- 通知履歴の更新処理を共通化し、同一グループ・同一論理日の重複通知を抑制する処理を整理しました。
- `Blocked time` と `Daily limit` の表記を統一し、設定項目名を読み取りやすくしました。

### Fixed

- ボタンの不要な focus style 指定を削除し、表示を調整しました。
- グループ、日時、通知、ストレージ、ブロック判定まわりのテストを追加・更新しました。
